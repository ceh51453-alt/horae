【Система памяти Horae】(Нижеприведенные примеры только для демонстрации; не вставляй их дословно в основной текст!)

═══ Ключевой принцип: управление изменениями ═══
★★★ Перед записью тега <horae> сначала определи, какие данные в этом ходе изменились по сути ★★★
  ① База сцены (time/location/characters/costume) → обязательна в каждом ходе
  ② Все остальные поля → строго следовать своим 【условиям срабатывания】; если изменений нет, строку полностью не писать
  ③ Для уже записанных NPC/предметов нет новой информации → вывод запрещен! Повтор неизменных данных = трата token
  ④ Изменились только отдельные поля → использовать инкрементальное обновление, писать только изменившееся
  ⑤ Первое появление NPC → обязательно писать и npc:, и affection:

═══ Формат тегов ═══
В конце каждого ответа обязательно указывать два тега:
<horae>
time:дата время (обязательно)
location:локация (обязательно. Для многоуровневых мест использовать разделитель ·, например "酒馆·大厅", "皇宫·王座间". Для одной и той же локации название должно быть всегда строго одинаковым)
atmosphere:атмосфера
characters:персонажи в сцене, через запятую (обязательно)
costume:角色名=описание одежды (обязательно, по строке на персонажа, объединять через ; запрещено)
item/item!/item!!:см. правила предметов (писать только при срабатывании)
item-:物品名 (удалять при расходе/утрате предмета. См. правила предметов, писать только при срабатывании)
affection:角色名=уровень симпатии (★ при первом появлении NPC обязателен начальный уровень; дальше обновлять только при изменении)
npc:角色名|外貌=性格@关系~扩展字段 (★ при первом появлении NPC обязательна полная информация; дальше обновлять только изменившееся)
agenda:日期|内容 (писать только при появлении новой задачи)
agenda-:内容关键词 (писать только при выполнении/утрате актуальности задачи, система автоматически удалит совпавшую задачу)
</horae>
<horaeevent>
event:重要程度|подробная сводка события (уровни: 一般/重要/关键)
</horaeevent>

═══ 【Правила написания сводки события (event)】═══
★ Цель: дать будущему ИИ "предысторию" без потерь. Текст должен быть конкретным и насыщенным, 80-150 слов.
★ Обязательно включить элементы (5W1H):
  ① Ключевое взаимодействие: кто кому что сделал/сказал? (конкретные действия или смысл ключевых реплик)
  ② Смена состояния/эмоций: какие тонкие изменения произошли в психологии, отношении или связях персонажей? (например, настороженность → доверие, тайная влюбленность)
  ③ Новая информация/результат: что сюжетно продвинулось в этом ходе? (какие улики получены, какие договоренности достигнуты, какие изменения случились)
  ④ Фоreshadowing/интрига (если есть): какие нерешенные вопросы остались?
★ Запрещены расплывчатые формулировки:
  ❌ Ошибка: "U и 艾伦 поговорили в таверне, хорошо провели время и договорились встретиться снова." (без деталей)
  ✅ Верно: "U в таверне расспросил 艾伦 о местонахождении торговца с черного рынка. Сначала 艾伦 насторожился, но после 10金币 от U раскрыл, что торговец появится завтра вечером у заброшенного причала. Отношение 艾伦 к U сменилось с настороженности на жадность. Они договорились действовать вместе завтра вечером."


═══ 【Предметы】Условия и правила ═══
Ссылайся на ID (#ID) в [物品清单] и строго решай, нужно ли выводить поле.

【Когда писать】(если выполнено хотя бы одно условие)
  ✦ Получен новый предмет → item:/item!:/item!!:
  ✦ Изменились количество/владелец/позиция/свойства существующего предмета → item: (только изменившаяся часть)
  ✦ Предмет израсходован/утрачен/закончился → item-:物品名
【Когда не писать】
  ✗ По предмету нет никаких изменений → запрещено выводить любые строки item
  ✗ Предмет лишь упомянут, но его состояние не изменилось → не писать

【Формат】
  Новый предмет: item:emoji物品名(数量)|描述=持有者@精确位置 (描述 можно опустить. Но если предмет имеет особый смысл, как подарок или памятный предмет, добавить 描述)
  Новый (важный): item!:emoji物品名(数量)|描述=持有者@精确位置 (для важного предмета описание обязательно: внешний вид + функция + источник)
  Новый (ключевой): item!!:emoji物品名(数量)|描述=持有者@精确位置 (ключевой предмет, описание обязательно подробное)
  Изменение существующего: item:emoji物品名(新数量)=新持有者@新位置 (обновлять только изменившееся; если без "|", сохраняется прежнее описание)
  Расход/утрата: item-:物品名

【Правила уровня поля】
  · 描述: фиксирует сущностные свойства предмета (внешний вид/функция/источник), для обычных предметов можно опустить, для важных/ключевых при первом появлении обязательно
    ★ Внешние признаки (цвет, материал, размер и т.п., для последующей согласованности описания)
    ★ Функция/назначение
    ★ Источник (кто дал/как получен)
       - Примеры (ниже примеры только для демонстрации, не копировать дословно в основной текст):
         - Пример1: item!:🌹永生花束|букет из темно-красных вечных роз, перевязан черной лентой, подарок C для U на День влюбленных=U@на столе в комнате U
         - Пример2: item!:🎫幸运十连抽券|бумажный купон с золотым свечением, стартовый бонус на одну десятикратную вытяжку в системном пуле=U@пространственное кольцо
         - Пример3: item!!:🏧位面货币自动兑换机|похоже на маленький ATM, может обменивать валюты разных измерений по текущему курсу=U@стойка таверны
  · 数量: для единичных предметов не писать (1)/(1 шт.)/(1 меч), скобки указывать только при наличии единицы измерения, например (5斤)(1L)(1箱)
  · 位置: должна быть точной фиксированной локацией
    ❌ перед кем-то на земле, у чьих-то ног, рядом с кем-то, пол, на столе
    ✅ пол зала таверны, на барной стойке ресторана, домашняя кухня, в рюкзаке, на столе в комнате U
  · Нельзя относить стационарную мебель и элементы здания к предметам
  · Временное одалживание ≠ передача принадлежности


Пример (жизненный цикл эля):
  Получение: item:🍺陈酿麦酒(50L)|эль из кладовой, кисловатый вкус=U@шкаф ингредиентов в задней кухне таверны
  Изменение количества: item:🍺陈酿麦酒(25L)=U@шкаф ингредиентов в задней кухне таверны
  Израсходован: item-:陈酿麦酒

═══ 【NPC】Условия и правила ═══
Формат: npc:名|外貌=性格@与{{user}}的关系~性别:值~年龄:值~种族:值~职业:值~生日:值
Разделители: | имя, = внешность/характер, @ отношение, ~ расширенные поля (key:value)

【Когда писать】(если выполнено хотя бы одно условие, выводить npc: строку этого NPC)
  ✦ Первое появление → полный формат, все поля + все ~расширенные поля (пол/возраст/раса/профессия), ничего не пропускать
  ✦ Постоянное изменение внешности (шрам, новая прическа, смена одежды) → писать только поле внешности
  ✦ Изменение характера (после крупного события) → писать только поле характера
  ✦ Изменение роли в отношении к {{user}} (например, гость → друг) → писать только поле отношения
  ✦ Получена новая информация об этом NPC (ранее неизвестные рост/вес и т.д.) → дописывать в соответствующие поля
  ✦ Изменилось само ~расширенное поле (например, профессия) → писать только изменившееся ~поле
【Когда не писать】
  ✗ NPC в сцене, но новой информации нет → не писать строку npc:
  ✗ NPC временно уходил и вернулся без изменений → не переписывать
  ✗ Перефразировать существующее описание синонимами/сокращениями → строго запрещено
    ❌ "肌肉发达/满身战斗伤痕" → "мускулистый/шрамы" (замена слов ≠ обновление)
    ✅ "肌肉发达/满身战斗伤痕/重伤" → "肌肉发达/满身战斗伤痕" (ранение зажило, устаревшее состояние удалено)

【Примеры инкрементального обновления】(на примере NPC 沃尔)
  Первое появление: npc:沃尔|серо-серебристая шерсть/зеленые глаза/рост 220см/боевые шрамы по всему телу=молчаливый тяжеловооруженный наемник@первый гость {{user}}~性别:男~年龄:约35~种族:狼兽人~职业:佣兵
  Только отношение: npc:沃尔|=@парень {{user}}
  Только добавить внешность: npc:沃尔|серо-серебристая шерсть/зеленые глаза/рост 220см/боевые шрамы по всему телу/бинт на левой руке
  Только характер: npc:沃尔|=больше не молчалив/иногда улыбается
  Только профессия: npc:沃尔|~职业:退役佣兵
(Важно: неизменившиеся поля и неизменившиеся ~расширенные поля вообще не писать! Система автоматически сохраняет прежние данные!)

【Поле дня рождения (необязательное расширение)】
  Формат: ~生日:yyyy/mm/dd или ~生日:mm/dd (если год неизвестен, указывать только месяц/день)
  ⚠ Заполнять только если дата рождения явно указана в настройках/описании персонажа. Угадывать и выдумывать запрещено.
  ⚠ Если нет явного источника, это поле не заполнять — оставить пустым для пользователя.

【Норма описания отношений】
  Обязательно указывать точное имя объекта: ❌гость ✅новый посетитель {{user}} / ❌кредитор ✅человек, у которого долговая расписка {{user}} / ❌арендодатель ✅арендодатель {{user}} / ❌парень ✅парень {{user}} / ❌благодетель ✅человек, спасший жизнь {{user}} / ❌обидчик ✅человек, который травит {{user}} / ❌тайный воздыхатель ✅человек, тайно влюбленный в {{user}} / ❌враг ✅человек, чей биологический отец был убит {{user}}
  Для зависимых отношений указывать имя владельца-NPC: ✅охотничий пёс Ивана; питомец гостя {{user}} / девушка Ивана; гость {{user}} / лучшая подруга {{user}}; жена Ивана / отчим {{user}}; отец Ивана / любовник {{user}}; младший брат Ивана / лучшая подруга {{user}}; любовница мужа {{user}}; третье лицо, вмешавшееся в брак {{user}} и Ивана

═══ 【Симпатия】Условия срабатывания ═══
Записывать только симпатию NPC к {{user}} (не записывать {{user}}). По одной строке на человека, комментарии после числа запрещены.

【Когда писать】
  ✦ Первое появление NPC → установить начальное значение по отношению (陌生0-20/熟人30-50/朋友50-70/恋人70-90)
  ✦ Взаимодействие привело к существенному изменению симпатии → affection:имя=новое общее значение
【Когда не писать】
  ✗ Симпатия не изменилась → не писать

═══ 【Задачи】Условия срабатывания ═══
【Когда писать (добавление)】
  ✦ В сюжете появилась новая договоренность/план/маршрут/задача/фоreshadowing → agenda:日期|内容
  Формат: agenda:дата фиксации|содержимое (для относительного времени в скобках указывать абсолютную дату)
  Пример: agenda:2026/02/10|艾伦邀请{{user}}情人节晚上约会(2026/02/14 18:00)
【Когда писать (удаление после выполнения) — крайне важно】
  ✦ Задача выполнена/утратила актуальность/отменена → обязательно отметить удаление через agenda-:
  Формат: agenda-:содержимое задачи (достаточно ключевых слов содержимого для автоудаления)
  Пример: agenda-:艾伦邀请{{user}}情人节晚上约会
  ⚠ Запрещено писать в формате agenda:содержимое(выполнено)! Обязательно использовать префикс agenda-:
  ⚠ Запрещено повторно вписывать уже существующую задачу
【Когда не писать】
  ✗ Существующие задачи без изменений → не дублировать каждый ход
  ✗ Задача выполнена → запрещено помечать выполнение через agenda: + скобки, нужно только agenda-:

═══ Правила формата времени ═══
Запрещены расплывчатые форматы вроде "Day 1"/"第X天". Нужно использовать конкретные календарные даты.
- Современность: год/месяц/день час:минута (например 2026/2/4 15:00)
- Исторический сеттинг: дата соответствующей эпохи (например 1920/3/15 14:00)
- Фэнтези/альтернативный мир: календарь соответствующего мира (например 霜降月第三日 黄昏)
- Если точное время определить невозможно, укажи разумное конкретное время; запрещены бессмысленные формулировки вроде "неизвестно" и "в некий день".

${systemPromptAddition}
═══ Финальное обязательное напоминание ═══
Конец твоего ответа обязан содержать оба тега: <horae>...</horae> и <horaeevent>...</horaeevent>.
Отсутствует хотя бы один тег = невалидно.

【Поля, обязательные в каждом ходе — пропуск любого = невалидно】
  ✅ time: ← текущая дата и время
  ✅ location: ← текущая локация
  ✅ atmosphere: ← атмосфера
  ✅ characters: ← все персонажи, присутствующие в сцене, через запятую (пропускать нельзя)
  ✅ costume: ← по одной строке описания одежды для каждого присутствующего персонажа
  ✅ event: ← важность|сводка события

【Дополнительно обязательно при первом появлении NPC — пропуск недопустим】
  ✅ npc:名|外貌=性格@关系~性别:值~年龄:值~种族:值~职业:值~生日:值 (день рождения писать только если известен; если неизвестен — не писать)
  ✅ affection:имя этого NPC=начальная симпатия (陌生0-20/熟人30-50/朋友50-70/恋人70-90)

Для указанных выше полей нет варианта "можно писать, можно не писать" — они обязательны.
