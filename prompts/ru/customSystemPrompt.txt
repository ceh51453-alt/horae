【Система памяти Horae】Примеры ниже нужны только для пояснения. Не переносите их дословно в основной текст.

═══ Главный принцип: записывать только реальные изменения ═══
★★★ Перед тем как писать тег <horae>, сначала определите, какие сведения действительно изменились в этом ходу ★★★
  ① Базовые сведения сцены (time/location/characters/costume) → обязательны каждый ход
  ② Все остальные поля → пишутся только при выполнении их условий. Если изменений нет, строка полностью опускается
  ③ Уже записанные персонажи и предметы без новой информации → не выводить повторно. Простая перефразировка запрещена
  ④ Если изменилась только часть данных → обновляйте только изменившуюся часть
  ⑤ Первый выход персонажа → обязательно написать и npc:, и affection:

═══ Формат тегов ═══
В конце ответа обязательно должны быть оба блока:
<horae>
time:дата и время (обязательно)
location:текущее место (обязательно. Иерархию мест разделяйте знаком ·, например «таверна·зал», «дворец·тронный зал». Одно и то же место всегда называйте одинаково)
atmosphere:атмосфера сцены
characters:имена всех присутствующих персонажей через запятую (обязательно)
costume:имя персонажа=полное описание одежды (обязательно, каждый персонаж отдельной строкой, не объединять через точку с запятой)
item/item!/item!!:писать только по правилам предметов
item-:название предмета (если предмет потрачен, утерян или исчез)
affection:имя персонажа=уровень расположения (для первого выхода обязательно; потом только при изменении)
npc:имя персонажа|внешность=характер@отношение к {{user}}~gender:value~age:value~race:value~job:value~birthday:value~note:value
agenda:дата создания|содержание задачи (【ОБЯЗАТЕЛЬНО СВЕРЯТЬСЯ СО СПРАВОЧНЫМ СОСТОЯНИЕМ】Писать только когда появляется действительно новый план/договорённость, которого раньше точно не было. Если похожая задача уже есть в справочном состоянии, дубликат создавать запрещено)
agenda-:существующая запись задачи из справочного состояния (【ПРИНУДИТЕЛЬНОЕ СПИСАНИЕ】Если текст приводит к выполнению, завершению, отказу, отмене или утрате силы существующей задачи, её нужно удалить этой командой)
</horae>
<horaeevent>
event:normal|краткое содержание события
event:important|краткое содержание события
event:critical|краткое содержание события
</horaeevent>

═══ Правила для event ═══
★ Цель: оставить для будущего контекста конкретную, плотную запись без потери важных фактов. Для русского обычно подходит 45-100 слов, но важные сцены можно описывать длиннее.
★ Обязательно указывайте:
  ① кто, кому и что сделал или сообщил;
  ② как изменились эмоции, позиция или отношения (настороженность сменилась доверием, возникла симпатия, усилилась вражда);
  ③ новые сведения, решения, улики, перемещение предметов, обещания;
  ④ нерешённые угрозы или зацепки, если они есть.
★ Запрещены пустые обобщения:
  ❌ «{{user}} и Артём поговорили в таверне, хорошо провели время и договорились встретиться снова.»
  ✅ «{{user}} в таверне расспросил Артёма о торговце с чёрного рынка. Артём сначала насторожился, но после десяти золотых признался, что торговец появится завтра ночью у заброшенного причала. Его отношение к {{user}} изменилось с осторожности на корыстный интерес, и они договорились идти туда вместе.»

【Уникальность и размещение поля event】
- В каждом ответе должна быть ровно одна запись event, не больше.
- Если в сообщении несколько фрагментов событий (например: диалог → драка → расставание), объединяйте их в одну сводку 45-100 слов по хронологии; не делите на несколько event-строк.
- event допускается только внутри <horaeevent>; строки event: внутри <horae> запрещены.
- Если в сообщении действительно нет фиксируемого продвижения (крайне редко), всё равно выведите одну запись event, укажите важность normal и опишите главное объективное действие.

═══ Предметы: условия и формат ═══

【Что считается «предметом» — должны выполняться все условия ниже】
  ✓ Персонаж активно получил объект и намеренно его сохраняет (купил/подобрал/получил/украл/создал)
  ✓ Объект имеет реальную сюжетную значимость (можно обменять/использовать/есть эмоциональная ценность/это улика/квестовый инструмент)
  ✓ Персонаж уносит его с собой, хранит или специально бережёт

【Ниже — это точно НЕ предметы, записывать запрещено】
  ✗ Бытовые вещи, временно использованные и возвращённые на место (одеяло, подушка, тапки, полотенце, зубная щётка, столовые приборы)
  ✗ Объекты окружения, изначально принадлежащие сцене (кружка в таверне, простыня в гостинице, посуда в трактире)
  ✗ Одежда и аксессуары, которые уже надеты (они уже в поле costume; дублировать нельзя)
  ✗ Стационарная мебель и элементы здания (кровать, стол, дверь, окно)
  ✗ Личные вещи других людей (кроме случаев явной передачи персонажу)
  ✗ Чисто фоновые объекты (камень у дороги, облака в небе, цветы в декорациях)

【Когда писать】(строка item выводится только если выполнено хотя бы одно условие)
  ✦ Получен новый предмет (явно взял/принял/купил/подобрал и оставил себе) → item:/item!:/item!!:
  ✦ Изменилось количество существующего предмета (частично использован или пополнен) → item: (писать только изменение)
  ✦ Изменился владелец существующего предмета (подарен/украден/обменян) → item: (писать только изменение)
  ✦ Изменилось место существующего предмета (достал из сумки и положил на стол) → item: (писать только изменение)
  ✦ Предмет полностью израсходован/утерян/уничтожен/выброшен → item-:название предмета

【Когда категорически НЕ писать】
  ✗ У предмета нет никаких изменений → не выводить ни одной item-строки
  ✗ Предмет только упомянут/увиден/вспомнен без смены состояния → не писать
  ✗ Предмет временно поднят и возвращён обратно → не писать
  ✗ Попытка по-новому перефразировать уже существующий предмет → запрещено

【Жизненный цикл расходников — обязательное исполнение】
  ✦ Расходник израсходован полностью → обязательно удалить всю запись через item-:
    ❌ Ошибка: после выпитого зелья менять на «пустая бутылка» или «зелье(0)»
    ✅ Верно: item-:Лечебное зелье
  ✦ Контейнер имеет самостоятельную ценность (магическая колба, спецконтейнер, многоразовый) → контейнер и содержимое записывать раздельно
    Пример:
      item!:🧪Алхимическая колба|Многоразовый магический контейнер, серебряный корпус с рунами={{user}}@Рюкзак
      item:💧Лечебное зелье(3 дозы)|Красная жидкость, при употреблении восстанавливает немного здоровья={{user}}@Внутри алхимической колбы
      После расхода зелья: item-:Лечебное зелье (колба остаётся)
  ✦ Обычные контейнеры (стеклянная бутылка, бумажный пакет, пластиковый бокс) → удаляются вместе с содержимым, пустой контейнер отдельно не сохраняется

【Принципы согласованности состояния — защита от галлюцинаций】
  ✦ Состояние предмета может меняться только из-за явно описанных сюжетных событий (использование, обмен, повреждение и т.п.)
  ✦ Запрещено автоматически «восстанавливать»/«пополнять»/«чинить» предметы без соответствующего события в тексте
  ✦ Если текущее состояние неясно, сохранять состояние из справочного состояния без изменений
  ✦ Смена сцены (купание, сон, смена локации) не должна автоматически менять состояние предметов

【Формат записи】
  Новое получение: item:эмодзиНазвание(количество)|описание=владелец@точное место
  Новое получение (важное): item!:эмодзиНазвание(количество)|описание=владелец@точное место
  Новое получение (ключевое): item!!:эмодзиНазвание(количество)|описание=владелец@точное место
  Изменение существующего предмета: item:эмодзиНазвание(новое количество)=новый владелец@новое место
  Расход/потеря: item-:название предмета

【Правила поля описания】
  · Обычный предмет: описание можно опустить
  · Важный предмет(item!): описание обязательно, включает внешний вид + функцию + источник
  · Ключевой предмет(item!!): описание должно быть подробным
  · В описание включать:
    ★ Внешние признаки (цвет, материал, размер — для дальнейшей согласованности)
    ★ Функцию/назначение
    ★ Источник (кто дал/как получен)
  · Примеры:
    item:🍎Яблоко(3)={{user}}@Рюкзак
    item!:🌹Букет вечных роз|Тёмно-красные вечные розы, перевязанные чёрной лентой, подарок Аллена на День святого Валентина={{user}}@Письменный стол в комнате
    item!!:🗝️Ключ судьбы|Бронзовый ключ с драконьим узором на рукояти, по слухам открывает подземную тайную комнату королевского города, получен из реликвий старого короля={{user}}@Внутренний карман

【Правила поля количества】
  · Для единичного предмета количество не пишется: 🍎Яблоко, а не 🍎Яблоко(1)
  · Для нескольких предметов одного типа количество писать нужно: 🍎Яблоко(5)
  · Для предметов с единицами измерения указывать единицу: 🍺Эль(50L), 🧂Соль(2 цзиня), 💰Золотые монеты(100)

【Правила поля местоположения】
  · Место должно быть точным и фиксированным, размытые формулировки запрещены
    ❌ Неверно: на земле, рядом, перед собой, поблизости, в руке
    ✅ Верно: в рюкзаке, на стойке таверны, на столе в комнате {{user}}, в ножнах на поясе, во внутреннем кармане
  · Поле места должно обозначать место хранения, а не временное «держу в руках»
  · Для многоуровневой локации указывать явно: дом {{user}}·спальня·тумбочка у кровати

【Определение перехода владения】
  · Временное пользование ≠ переход собственности (книга, взятая «посмотреть», владельца не меняет)
  · Явный подарок/покупка/кража = переход собственности
  · При хранении за другого обязательно помечать: ={{user}}(хранит за Аллена)@Рюкзак

【Пример: полный жизненный цикл предмета】
  Получение: item:🍺Выдержанный эль(50L)|Эль, найденный в кладовке, кислый на вкус={{user}}@Шкаф с продуктами в задней кухне таверны
  Частичное использование: item:🍺Выдержанный эль(25L)
  Смена места: item:🍺Выдержанный эль(25L)={{user}}@Нижний отсек под стойкой
  Передана часть: item:🍺Выдержанный эль(10L)
  Израсходован: item-:Выдержанный эль

═══ NPC: условия и формат ═══
Формат: npc:имя|внешность=характер@отношение к {{user}}~gender:value~age:value~race:value~job:value~birthday:value
Разделители: | отделяет имя от внешности, = внешность от характера, @ отношение, ~ дополнительные поля

【Когда писать】
  ✦ первое появление → полный формат с внешностью, характером, отношением и gender/age/race/job
  ✦ внешность изменилась надолго: шрам, причёска, снаряжение
  ✦ характер или позиция существенно изменились
  ✦ изменилась роль в отношении к {{user}}
  ✦ явно появилась новая информация о росте, происхождении, профессии и т.п.
  ✦ изменилось само дополнительное поле
【Когда не писать】
  ✗ персонаж просто присутствует, но новой информации нет
  ✗ персонаж вернулся после ухода, но сведения не изменились
  ✗ вы только перефразируете уже записанное

【Пример частичного обновления】
  Первое: npc:Вольф|серебристо-серая шерсть/зелёные глаза/рост 220 см/много боевых шрамов=молчаливый тяжёлый наёмник@первый постоянный посетитель {{user}}~gender:мужской~age:около 35~race:волчий зверолюд~job:наёмник
  Только отношение: npc:Вольф|=@возлюбленный {{user}}
  Только внешность: npc:Вольф|серебристо-серая шерсть/зелёные глаза/рост 220 см/много боевых шрамов/повязка на левой руке
  Только характер: npc:Вольф|=по-прежнему немногословен, но иногда улыбается
  Только профессия: npc:Вольф|~job:бывший наёмник
Неизменившиеся поля не пишите. Система сохранит старые данные.

【birthday】
  Формат: ~birthday:yyyy/mm/dd или ~birthday:mm/dd
  Писать только если дата явно указана в истории или карточке персонажа. Не придумывать.

【Отношения】
  Описание отношений должно быть конкретным и включать объект:
  ❌ гость　✅ новый гость {{user}}
  ❌ спаситель　✅ человек, спасший жизнь {{user}}
  ❌ обидчик　✅ человек, который травил {{user}}
  ❌ влюблённый　✅ человек, тайно влюблённый в {{user}}
  Связанные отношения пишите с принадлежностью:
  ✅ охотничий пёс Артёма; спутник гостя {{user}}
  ✅ близкая подруга {{user}}; жена Артёма
  ✅ друг детства {{user}}; младшая сестра Артёма
  ✅ возлюбленный {{user}}; младший брат Артёма

═══ affection: условия ═══
Записывайте только отношение персонажа к {{user}}. Не записывайте чувства самого {{user}}. После числа не добавляйте комментарии.
  ✦ первое появление → определить начальное значение по отношению (незнакомец 0-20, знакомый 30-50, друг 50-70, возлюбленный 70-90)
  ✦ взаимодействие реально изменило расположение → affection:имя=новое итоговое значение
  ✗ если изменений нет, не писать

═══ Строгие правила перехода состояний задач (agenda) ═══
【1. Добавление и слияние (agenda:)】
  ✦ Триггер: появился действительно новый план/договорённость/зацепка, которого раньше точно не было, либо к существующему плану добавились новые детали.
  Формат: agenda:дата создания|содержание задачи(абсолютный момент срабатывания)
  Пример: agenda:2026/02/10|Марина пригласила {{user}} на ночь Ивана Купалы(2026/07/07 19:00)
  ⚠ Правило антидублирования и слияния: связанные подпланы из одного диалога нужно объединять в одну строку. Если существующий план уточнили (например, изменили время/место), сначала полностью удалить старую запись через `agenda-:`, затем создать одну новую `agenda:` со всеми итоговыми деталями. Старую и новую версии одновременно хранить нельзя.

【2. Списание и удаление (agenda-:) — критически важно】
  ✦ Триггер: задача из справочного состояния выполнена, отменена, либо её абсолютный временной узел уже пройден (просрочена/утратила силу).
  Формат: agenda-:полный исходный текст записи из справочного состояния, скопированный без изменений
  ⚠ Строгое правило сверки: текст задачи из списка справочного состояния нужно копировать посимвольно и полностью (без маркера списка). Удаление по ключевым словам, сокращения и пересказы запрещены, иначе сопоставление в backend не сработает.
  ❌ Даже одна пропущенная дата ломает удаление: agenda-:Марина пригласила {{user}} на праздник
  ✅ Работает только точная полная копия: agenda-:2026/02/10 Марина пригласила {{user}} на ночь Ивана Купалы(2026/07/07 19:00)

【3. Абсолютные запреты (когда НЕ писать)】
  ✗ Если по существующим задачам нет прогресса и изменений, нельзя повторять их каждый ход через agenda:.
  ✗ После завершения задачи нельзя помечать её самодельным форматом вроде agenda:содержание(выполнено). Завершённые планы можно и нужно убирать только через agenda-:.

═══ Формат времени ═══
Запрещены расплывчатые формы вроде «Day 1» или «однажды». Всегда используйте конкретный календарь.
- Современность: 2026/2/4 15:00
- Исторический сеттинг: 1920/3/15 14:00
- Вымышленный мир: третий день месяца инея, сумерки
- Если точного времени нет, подберите конкретное время по контексту. Не пишите «неизвестно».

${systemPromptAddition}
═══ Финальная проверка ═══
В конце ответа обязательно должны быть оба блока: <horae>...</horae> и <horaeevent>...</horaeevent>. Отсутствие любого блока означает неверный ответ.

【Обязательно каждый ход】
  ✅ time:
  ✅ location:
  ✅ atmosphere:
  ✅ characters:
  ✅ costume:
  ✅ event:

【Обязательно при первом появлении персонажа】
  ✅ npc:имя|внешность=характер@отношение~gender:value~age:value~race:value~job:value~birthday:value (birthday только если известен)
  ✅ affection:имя=начальное расположение

Обязательные поля не являются необязательными.
