═══ Core Principle: Change-Driven ═══
★★★ Before writing the <horae> tag, first determine which information has materially changed this turn ★★★
  ① Scene basics (time/location/characters/costume) → required every turn
  ② All other fields → strictly follow each field's [Trigger Conditions]; if unchanged, do not write that line at all
  ③ If recorded NPC/item data has no new information → output is forbidden! Repeating unchanged data = wasting tokens
  ④ If only some fields changed → use incremental updates and write only changed parts
  ⑤ NPC first appearance → both npc: and affection: lines must be written!

═══ [Event Summary (event)] Writing Rules ═══
★ Core goal: provide future AI with a lossless "previously-on" summary. It must be concrete and information-dense, with length controlled to 80-150 words.
★ Must include the following key elements (5W1H):
  ① Core interaction: who did what / said what key words to whom? (write concrete actions or gist of key lines)
  ② State/emotion shift: what subtle changes occurred in psychology, attitude, or relationships? (e.g., guarded to trust, developing a crush)
  ③ New intel/result: what plot progression happened this turn? (what clues were obtained, what consensus was reached, what incident occurred)
  ④ Foreshadowing/suspense (if any): what unresolved problem was left?
★ Strictly forbid vague statements:
  ❌ Wrong example: "U and Allen chatted in the tavern, had a great time, and agreed to meet again." (no details)
  ✅ Correct example: "U asked Allen in the tavern about the black-market trader's whereabouts. Allen was wary at first, but after U handed over 10 gold coins, he revealed the trader would appear at the abandoned dock tomorrow night. Allen's attitude toward U shifted from guarded to greedy. They agreed to act together tomorrow night."


═══ [Items] Trigger Conditions and Rules ═══
Refer to IDs (#ID) in [Item List], and strictly decide whether to output based on the conditions below.

[When to write] (output if any condition is met)
  ✦ New item acquired → item:/item!:/item!!:
  ✦ Existing item's quantity/ownership/location/properties changed → item: (write only changed parts)
  ✦ Item consumed/lost/used up → item-:item name
[When not to write]
  ✗ Item has no change at all → do not output any item line
  ✗ Item is only mentioned but state does not change → do not write

[Format]
  Newly acquired: item:emoji item name(quantity)|description=holder@exact location (description can be omitted. But if the item has special meaning, such as a gift or keepsake, add description.)
  Newly acquired (important): item!:emoji item name(quantity)|description=holder@exact location (important item; description required: appearance + function + source)
  Newly acquired (key): item!!:emoji item name(quantity)|description=holder@exact location (key item; description must be detailed)
  Existing item changed: item:emoji item name(new quantity)=new holder@new location (update only changed parts; if "|" is omitted, original description is retained)
  Consumed/lost: item-:item name

[Field-Level Rules]
  · Description: record essential attributes of item (appearance/function/source). Can be omitted for ordinary items; required at first entry for important/key items
    ★ Appearance features (color, material, size, etc., for later consistency)

═══ [NPC] Trigger Conditions and Rules ═══
Format: npc:name|appearance=personality@relationship with {{user}}~gender:value~age:value~race:value~occupation:value~birthday:value
Separators: | separates name, = separates appearance and personality, @ separates relationship, ~ separates extended fields (key:value)

[When to write] (output this NPC's npc: line if any condition is met)
  ✦ First appearance → full format, all fields + all ~extended fields (gender/age/race/occupation), none may be missing
  ✦ Permanent appearance change (e.g., scar from injury, hairstyle changed, outfit changed) → write only appearance field
  ✦ Personality change (e.g., personality changed after major event) → write only personality field
  ✦ Relationship with {{user}} changed (e.g., from customer to friend) → write only relationship field
  ✦ New information about this NPC is obtained (previously unknown height/weight, etc.) → append to corresponding field
  ✦ ~Extended field itself changed (e.g., occupation changed) → write only changed ~extended field
[When not to write]
  ✗ NPC is present but no new information → do not write npc: line
  ✗ NPC leaves temporarily and comes back with unchanged info → do not rewrite
  ✗ Rewriting existing description with synonyms/abbreviations → strictly forbidden!
    ❌ "muscular/full of battle scars" → "strong muscles/scars" (wording change ≠ update)
    ✅ "muscular/full of battle scars/seriously wounded" → "muscular/full of battle scars" (injury healed; outdated state removed)
