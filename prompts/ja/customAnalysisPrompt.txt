【出力形式】
<horae>

agenda-:参照状態に既存の予定エントリ（【強制消込】分析対象テキストにより参照状態の予定が実行・完了・放棄・取消・失効した場合、必ずこの指令で削除）

<horaeevent>

event:important|出来事の要約

【発動条件】
· 物品（厳格フィルタ）：
  【物品とみなす条件（すべて必須）】
    ✓ キャラが能動的に入手し、保持する意思がある（購入・拾得・受領・窃取・製作）
    ✓ 物語上の意味がある（取引可能／使用可能／感情的価値／手掛かり／武器・装備）
    ✓ 持ち運ぶ、または意図的に保管する対象である

  【以下は記録禁止】
    ✗ 一時使用の日用品（毛布、スリッパ、タオル、食器、椅子）
    ✗ 場所に元からある環境物（酒場のコップ、宿のシーツ、他人宅の家具）
    ✗ 着用中の服装（costumeに記録済み）
    ✗ 固定家具・建築設備
    ✗ 飲食物（鍵アイテム／毒物／特殊料理を除く）
    ✗ 参照状態に既に存在し、このターンで変化がない物品

  【消耗品処理（強制）】
    ✦ 使い切り／飲み切り／食べ切り → 必ず item-: で項目ごと削除し、「空き瓶」「空箱」に書き換えない
    ✦ 一部消費 → item: で数量更新（例：item:💊鎮痛薬(2)=U@ポケット）
    ✦ 通常容器は中身と一緒に削除し、空容器を単独保持しない
    ✦ 特殊容器（魔法瓶、高価な箱）だけは単独記録可

  【状態整合性】
    ✦ 物品状態は、このターンで明示された出来事によってのみ変化できる
    ✦ このターンで言及されない物品 → item行を一切書かず、参照状態を維持
    ✦ 「自動回復」「自動補充」「自動出現」を捏造しない

  【形式】新規取得は完全記述、既存物品は変化部分のみ。単品に(1)は付けない。絵文字接頭辞は必須。
· 人物：初登場は外見、性格、関係、gender/age/race/job を含める。以後は変化した項目だけ書く。
· 好感度：人物が{{user}}に向ける好感度だけを書く。初対面0〜20、顔見知り30〜50、友人50〜70、恋人70〜90を目安にする。
· 予定の状態遷移（最重要。以下のコードレベル規則を厳守）：
  【履歴ゴミの強制清掃】[参照状態] の [予定] 一覧を確認。分析対象テキストで予定が実行/取消された、または現在時刻がその予定の時刻ノードを越えた場合（本文に未言及でも失効扱い）、必ず `agenda-:` で消し込む。
  🔴 消込構文は厳格：`agenda-:` の後ろには元の予定エントリ全文を【一字一句そのまま】コピーする（先頭の箇条書き記号は含めない）。省略・言い換え・要約は禁止。
  消込例：agenda-:1988/09/29 山を下りて正式な夕食を取る(1988/09/29 18:00以降)
  【統合上書きと重複防止】新しい `agenda:` を作る前に既存 [予定] を比較する。
  1. 再言及のみ：計画に変化がなければ再出力禁止。
  2. 詳細更新：同一計画に新詳細が追加されたら、関連する旧エントリを `agenda-:` で全削除してから、新旧詳細を統合した最終版を `agenda:` 1行で再作成する。新旧の共存禁止。
  3. 一括統合：同一ターン/同一対話で生じた複数計画は1行に統合する。複数 `agenda:` 行への分割禁止。
  新規例：agenda:1988/09/29|明朝9時に起床して買い物に行き、制服着用を厳禁にする(1988/09/30 09:00)
· event は <horaeevent> 内に置く。<horae> 内に入れない。

【event 要約】
· 90〜180字程度を目安に、誰が誰に何をしたか、何を伝えたか、感情や関係がどう変わったか、何の手掛かりや結果が生まれたかを書く。
· 抽象的な言い方は禁止。
  ❌ 「{{user}}と蓮は話し合い、関係が少し良くなった。」
  ✅ 「{{user}}は廃駅で蓮に失踪した妹の手帳を見せ、蓮は疑いを残しつつも、手帳の暗号が旧市街の倉庫番号を示すと認めた。蓮は{{user}}を単なる依頼人ではなく協力者として扱い始めた。」
· 物語の続きの創作を厳禁：event の記述は【分析対象テキスト】の最後の明示的な動作/会話で必ず止める。
  原文にない後続の動作、返答、退場、締めの描写を補ってはいけない。論理的に「起こりそう」でも禁止。
  ❌ 誤例（原文が「ヒロインがミルクティーを欲しいと言う」で終わる）:
     「……主人公が何を飲むか尋ね、ヒロインがミルクティーと答え、主人公はうなずいて買いに行く。」
     （「うなずいて買いに行く」は原文にない続き）
  ✅ 正例:
     「……主人公が何を飲むか尋ね、ヒロインがミルクティーと答える。」
     （原文の最後の文で止め、続きは補わない）

【event フィールドの一意性と配置】
- 返信ごとに event は必ず1件のみ。2件以上は禁止。
- この投稿に複数の出来事片（例：会話→戦闘→別れ）がある場合でも、90〜180字の要約1件に時系列で統合し、複数 event に分割しない。
- event は <horaeevent> 内にのみ記載し、<horae> 内に event: 行を出してはいけない。
- 記録すべき進展が本当にない場合（極めて稀）でも、event を1件出力し、重要度は normal とし、最も主要な客観動作を書く。
