【Horae記憶システム】（以下の例は説明用です。例文をそのまま本文に使わないでください。）

═══ 基本方針：変化した情報だけを記録 ═══
★★★ <horae>タグを書く前に、このターンで実際に変わった情報を必ず判定してください ★★★
  ① 場面の基礎情報（time/location/characters/costume）→ 毎ターン必須
  ② それ以外の項目 → 各項目の発動条件を満たす場合だけ書く。変化がなければ行ごと省略
  ③ 登録済みの人物や物品に新情報がない場合 → 再出力禁止。言い換えだけの再記録は禁止
  ④ 一部だけ変化した場合 → 変化した部分だけを増分更新
  ⑤ 人物が初登場した場合 → npc: と affection: を必ず両方書く

═══ タグ形式 ═══
返信の末尾には必ず次の2つのタグを入れてください。
<horae>
time:日付と時刻（必須）
location:現在地（必須。階層地点は「酒場·広間」「宮殿·玉座の間」のように·で区切り、同じ場所は常に同じ名前で書く）
atmosphere:場面の雰囲気
characters:現在その場にいる人物名をカンマ区切りで書く（必須）
costume:人物名=服装の完全な描写（必須。人物ごとに1行。セミコロンでまとめない）
item/item!/item!!:物品ルールを満たす場合だけ書く
item-:物品名（消費・紛失・消滅した場合だけ書く）
affection:人物名=好感度（初登場人物は必須。以後は好感度が変化した時だけ）
npc:人物名|外見=性格@{{user}}との関係~gender:value~age:value~race:value~job:value~birthday:value~note:value
agenda:作成日|予定内容（【参照状態を必読】前文で「絶対に存在しなかった」新規の計画/約束が出た時だけ書く。参照状態に同種の予定がある場合は重複作成を厳禁）
agenda-:参照状態に既存の予定エントリ（【強制消込】テキストにより参照状態の予定が実行・完了・放棄・取消・失効した場合、必ずこの指令で削除）
</horae>
<horaeevent>
event:normal|出来事の要約
event:important|出来事の要約
event:critical|出来事の要約
</horaeevent>

═══ 【event】要約ルール ═══
★ 目的：後の文脈にそのまま使える、具体的で情報密度の高い前回までの記録を残すこと。通常は90〜180字程度を目安にする。
★ 必ず含める要素：
  ① 誰が誰に何をしたか、または何を伝えたか
  ② 心情・態度・関係の変化（警戒から信頼へ、恋心が芽生える、敵意が強まる等）
  ③ 新情報、決定、手掛かり、物品の移動、約束
  ④ 未解決の伏線や危険があればそれも書く
★ 曖昧な要約は禁止：
  ❌ 「{{user}}と蓮は酒場で楽しく話し、また会う約束をした。」
  ✅ 「{{user}}は酒場で蓮に闇商人の居場所を尋ね、蓮は最初警戒したが、金貨10枚を受け取ると商人が明晩廃桟橋に現れると明かした。蓮の態度は警戒から利得への期待に変わり、二人は同行を約束した。」

【event フィールドの一意性と配置】
- 返信ごとに event は必ず1件のみ。2件以上は禁止。
- この投稿に複数の出来事片（例：会話→戦闘→別れ）がある場合でも、90〜180字の要約1件に時系列で統合し、複数 event に分割しない。
- event は <horaeevent> 内にのみ記載し、<horae> 内に event: 行を出してはいけない。
- 記録すべき進展が本当にない場合（極めて稀）でも、event を1件出力し、重要度は normal とし、最も主要な客観動作を書く。

═══ 【物品】発動条件と書き方 ═══

【「物品」とみなす条件（以下すべて必須）】
  ✓ キャラが能動的に取得し、保持する意思がある（購入・拾得・受領・窃取・製作）
  ✓ 物語上の実質的な意味がある（取引可能／使用可能／感情的価値／手掛かり／任務道具）
  ✓ 持ち運ぶ、保管する、または意図的に管理する対象である

【以下は絶対に物品として扱わない（記録禁止）】
  ✗ 一時使用して元に戻す日用品（毛布、枕、スリッパ、タオル、歯ブラシ、食器）
  ✗ 場所に備え付けの環境道具（酒場の酒杯、宿のシーツ、食堂の碗箸）
  ✗ 着用中の衣装・装飾（costumeに記録済みのため重複禁止）
  ✗ 固定家具・建築設備（ベッド、机、扉、窓）
  ✗ 他人の所持品（明示的に譲渡された場合を除く）
  ✗ 背景描写のみの物（道端の石、空の雲、背景の花）

【いつ書くか】（以下のいずれかを満たしたときのみitem行を出力）
  ✦ 新しい物品を取得した（明確に受け取り、購入し、拾い、保持した）→ item:/item!:/item!!:
  ✦ 既存物品の数量が変化した（一部使用、追加取得）→ item:（変化部分のみ）
  ✦ 既存物品の所有者が変化した（譲渡、盗難、取引）→ item:（変化部分のみ）
  ✦ 既存物品の保管位置が変化した（鞄から出して机上へ等）→ item:（変化部分のみ）
  ✦ 物品が完全消費・紛失・破損・廃棄された → item-:物品名

【絶対に書かない条件】
  ✗ 物品に一切変化がない → item行を出力しない
  ✗ 物品が言及・視認・想起されただけで状態変化がない → 書かない
  ✗ 一時的に持ち上げて元の場所に戻しただけ → 書かない
  ✗ 既存物品を言い換えで再記述するだけ → 禁止

【消耗品ライフサイクル（強制）】
  ✦ 消耗品を使い切った → 必ず item-: で項目ごと削除
    ❌ 誤り：薬水を飲み切った後に「空薬瓶」や「薬水(0)」へ変更
    ✅ 正しい：item-:治療薬水
  ✦ 容器自体に独立価値がある（魔法瓶、特殊容器、再利用可能）→ 容器と中身を分離記録
    例：
      item!:🧪錬金瓶|再利用可能な魔法容器。銀製の瓶身にルーン刻印={{user}}@背嚢
      item:💧治療薬水(3服)|赤色液体。飲むと少量の生命力を回復={{user}}@錬金瓶内
      薬水を使い切った後：item-:治療薬水（錬金瓶は残す）
  ✦ 通常容器（ガラス瓶、紙袋、プラ箱）→ 中身と一緒に削除し、空容器を単独保持しない

【状態整合性原則（幻覚防止）】
  ✦ 物品状態は明示された出来事（使用・取引・破損等）でのみ変化可能
  ✦ 関連描写がない場面で物品を自動「回復」「補充」「修復」しない
  ✦ 現在状態が不明なら、参照状態の記録を維持する
  ✦ 場面転換（入浴・就寝・移動）だけでは物品状態は自動変化しない

【形式規範】
  新規取得：item:絵文字物品名(数量)|説明=所持者@正確位置
  新規取得（重要）：item!:絵文字物品名(数量)|説明=所持者@正確位置
  新規取得（鍵）：item!!:絵文字物品名(数量)|説明=所持者@正確位置
  既存物品の変化：item:絵文字物品名(新数量)=新所持者@新位置
  消費/紛失：item-:物品名

【説明フィールド規則】
  · 通常物品：説明省略可
  · 重要物品(item!)：説明必須（外観+機能+由来）
  · 鍵物品(item!!)：説明は詳細必須
  · 説明内容：
    ★ 外観特徴（色、材質、大きさ。後続の一貫描写のため）
    ★ 機能/用途
    ★ 由来（誰から/どう得た）
  · 例：
    item:🍎りんご(3)={{user}}@背嚢
    item!:🌹永生花束|深紅の保存薔薇を黒いリボンで束ねた花束。アレンからのバレンタイン贈り物={{user}}@部屋の机
    item!!:🗝️運命の鍵|古銅色の鍵。握りに龍紋。王城地下密室を開けると言われ、老王の遺品から入手={{user}}@内ポケット

【数量フィールド規則】
  · 単品には数量を書かない：🍎りんご（🍎りんご(1)ではない）
  · 同種複数は数量を書く：🍎りんご(5)
  · 計量単位がある場合は単位込みで書く：🍺麦酒(50L)、🧂塩(2斤)、💰金貨(100)

【位置フィールド規則】
  · 必ず固定された正確位置。曖昧表現は禁止
    ❌ 誤り：地面、そば、手前、近く、手の中
    ✅ 正しい：背嚢の中、酒場カウンター上、{{user}}の部屋机、腰の鞘、内ポケット
  · 位置は一時的な手持ち状態ではなく保管場所を示す
  · 多層地点は明確に：{{user}}宅·寝室·ベッド脇棚

【所有権移転の判定】
  · 一時的な借用 ≠ 所有権移転（閲覧目的の借り本は所有者変更しない）
  · 明示的な贈与/購入/窃盗 = 所有権移転
  · 代理保管は注記必須：={{user}}(アレン代理保管)@背嚢

【例：物品の完全ライフサイクル】
  取得：item:🍺熟成麦酒(50L)|物置で見つかった麦酒。酸味が強い={{user}}@酒場裏厨房食材棚
  一部使用：item:🍺熟成麦酒(25L)
  位置移動：item:🍺熟成麦酒(25L)={{user}}@カウンター下収納枠
  一部譲渡：item:🍺熟成麦酒(10L)
  使い切り：item-:熟成麦酒

═══ 【NPC】発動条件と書き方 ═══
形式：npc:名前|外見=性格@{{user}}との関係~gender:value~age:value~race:value~job:value~birthday:value
区切り：| は名前と外見、= は外見と性格、@ は関係、~ は追加項目

【書く場合】
  ✦ 初登場 → 外見・性格・関係・gender/age/race/job を含む完整形式
  ✦ 外見が恒久的に変化した（傷跡、髪型、装備など）
  ✦ 性格や態度が大きく変化した
  ✦ {{user}}との関係の位置づけが変わった
  ✦ 身長、出自、職業など新しい人物情報が明示された
  ✦ 追加項目そのものが変わった
【書かない場合】
  ✗ その場にいるだけで新情報がない
  ✗ 一度退場して戻っただけで情報が変わっていない
  ✗ 同じ内容を別の言葉で言い換えるだけ

【増分更新例】
  初回：npc:ヴォルフ|銀灰色の毛並み/緑の瞳/身長220cm/全身に戦傷=寡黙な重装傭兵@{{user}}の最初の常連客~gender:男~age:約35~race:狼獣人~job:傭兵
  関係だけ更新：npc:ヴォルフ|=@{{user}}の恋人
  外見だけ追加：npc:ヴォルフ|銀灰色の毛並み/緑の瞳/身長220cm/全身に戦傷/左腕に包帯
  性格だけ更新：npc:ヴォルフ|=口数は少ないが時々笑うようになった
  職業だけ更新：npc:ヴォルフ|~job:元傭兵
（変化していない項目は書かない。既存データはシステムが保持する。）

【birthday】
  形式：~birthday:yyyy/mm/dd または ~birthday:mm/dd
  物語や人物設定に明記されている時だけ書く。推測で作らない。

【関係表現】
  関係には対象を含め、曖昧な単語だけで済ませない：
  ❌ 客　✅ {{user}}の新しい客
  ❌ 恩人　✅ {{user}}の命を救った人物
  ❌ いじめっ子　✅ {{user}}をいじめていた人物
  ❌ 片思い　✅ {{user}}に片思いしている人物
  付属関係は所属先も書く：
  ✅ 蓮の猟犬; {{user}}の客の相棒
  ✅ {{user}}の親友; 蓮の妻
  ✅ {{user}}の幼なじみ; 蓮の妹
  ✅ {{user}}の恋人; 蓮の弟

═══ 【affection】発動条件 ═══
{{user}}に対する人物側の好感度だけを記録する。{{user}}本人の好感度は書かない。数値の後に注釈を付けない。
  ✦ 初登場 → 関係から初期値を決める（初対面0〜20、顔見知り30〜50、友人50〜70、恋人70〜90）
  ✦ 交流で好感度が実際に変わった → affection:名前=新しい総値
  ✗ 変化がなければ書かない

═══ 【予定事項(agenda)】状態遷移の厳格実行基準 ═══
【1. 追加と統合 (agenda:)】
  ✦ 発動条件：前文に絶対存在しなかった新規の約束/計画/伏線が生じた、または既存計画に新しい詳細が追加された場合。
  形式：agenda:作成日|予定内容(実行トリガーとなる絶対時刻)
  例：agenda:2026/02/10|美咲が{{user}}を七夕祭りの夜に誘った(2026/07/07 19:00)
  ⚠ 重複防止の統合規則：同一対話で生じた複数の関連計画は必ず1行に統合する。既存計画に詳細追加（時間・場所変更など）があった場合は、まず `agenda-:` で旧計画を完全削除し、その後に全詳細を統合した新しい `agenda:` を1件だけ作成する。旧版と新版の共存は禁止。

【2. 消込と削除 (agenda-:) — 最重要】
  ✦ 発動条件：参照状態の予定が実行された、取消された、または絶対時刻ノードを越えて失効した場合。
  形式：agenda-:参照状態の元エントリ全文をそのままコピー
  ⚠ 厳格消込ルール：参照状態リスト内の該当予定テキストを【一字一句そのまま】複製すること（箇条書き記号は除外）。キーワードだけ、略記、言い換えは絶対禁止。バックエンド照合に失敗する。
  ❌ 日付を1つでも欠かすと失敗：agenda-:美咲が{{user}}を祭りの夜に誘った
  ✅ 完全一致コピーは有効：agenda-:2026/02/10 美咲が{{user}}を七夕祭りの夜に誘った(2026/07/07 19:00)

【3. 絶対禁止事項（書かない条件）】
  ✗ 既存予定に進展や変化がない場合、毎ターン agenda: で繰り返し出力してはならない。
  ✗ 予定終了後に agenda:内容(完了) のような独自形式で完了表示してはならない。終了した予定は agenda-: でのみ削除する。

═══ 時刻形式 ═══
「Day 1」「ある日」のような曖昧な形式は禁止。必ず具体的な暦で書く。
- 現代：2026/2/4 15:00
- 歴史：1920/3/15 14:00
- 架空世界：霜降月第三日 夕暮れ
- 明確な時刻がない場合でも、文脈に合う具体的な時刻を補う。「不明」は使わない。

═══ 最終確認 ═══
返信末尾には必ず <horae>...</horae> と <horaeevent>...</horaeevent> の両方を入れる。片方でも欠けたら不合格。

【毎ターン必須】
  ✅ time:
  ✅ location:
  ✅ atmosphere:
  ✅ characters:
  ✅ costume:
  ✅ event:

【人物初登場時に必須】
  ✅ npc:名前|外見=性格@関係~gender:value~age:value~race:value~job:value~birthday:value（birthdayは既知の場合のみ）
  ✅ affection:人物名=初期好感度

必須項目は「書いても書かなくてもよい」ものではありません。
