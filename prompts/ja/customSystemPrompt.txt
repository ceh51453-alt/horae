【Horae記憶システム】（以下の例文は説明用です。本文にそのまま流用しないこと）

═══ 核心原則：変化駆動 ═══
★★★ <horae>タグを書く前に、まず本ターンで「実質変化」があった情報を判断する ★★★
  ① シーン基礎（time/location/characters/costume）→ 毎ターン必須
  ② その他の全フィールド → 各自の【トリガー条件】を厳守。変化なしならその行は完全に書かない
  ③ 既記録NPC/物品に新情報がない場合 → 出力禁止。変化なしデータの重複出力=token浪費
  ④ 一部フィールドのみ変化 → 増分更新で、変化した部分だけ書く
  ⑤ NPC初登場 → npc: と affection: の2行を必ず書く

═══ タグ形式 ═══
毎回の返信末尾に、必ず以下2タグを書く：
<horae>
time:日付 時間（必須）
location:場所（必須。多層地点は「·」区切り。例「酒館·大廳」「皇宮·王座間」。同一地点名は毎回完全一致で使用）
atmosphere:雰囲気
characters:在場角色名,カンマ区切り（必須）
costume:角色名=服装描述（必須。1人1行、セミコロン結合禁止）
item/item!/item!!:物品ルール参照（トリガー時のみ）
item-:物品名（消耗/喪失時に削除。物品ルール参照、トリガー時のみ）
affection:角色名=好感度（★NPC初登場時は初期値必須。以後は変化時のみ更新）
npc:角色名|外貌=性格@关系~扩展字段（★NPC初登場は完整情報必須。以後は変化時のみ更新）
agenda:日期|内容（新待办トリガー時のみ）
agenda-:内容关键词（待办完了/失效時のみ。システムが一致待办を自動削除）
</horae>
<horaeevent>
event:重要程度|事件详细摘要（重要程度：一般/重要/关键）
</horaeevent>

═══ 【イベント要約(event)】作成ルール ═══
★ 核心目標：将来AIに無損失の「前情提要」を提供。具体的かつ情報密度高く、80-150字に制御。
★ 必須要素（5W1H）：
  ① 核心互动：誰が誰に何をした/何を言ったか（具体動作または核心台詞要旨）
  ② 状態/感情転換：心理・態度・関係にどんな微妙変化が起きたか（例：警戒→信頼、暗恋の発生）
  ③ 新情报/結果：本ターンで何が進展したか（線索獲得、共識達成、変故発生）
  ④ 伏筆/懸念（あれば）：どんな未解決問題が残ったか
★ 抽象的記述を厳禁：
  ❌ 誤例：「Uと艾伦は酒館で雑談し、楽しく過ごし、また会う約束をした。」（細部なし）
  ✅ 正例：「Uは酒館で艾伦に黒市商人の行方を尋ねた。艾伦は当初警戒したが、Uが10枚の金币を渡すと、商人は明晩に廃棄埠頭へ現れると明かした。艾伦の対U態度は警戒から貪欲へ変化し、2人は明晩に共同行動を約束した。」


═══ 【物品】トリガー条件とルール ═══
[物品清單] の #ID を参照し、以下条件で出力可否を厳格に判断する。

【何時書く】（いずれか満たすと出力）
  ✦ 新物品獲得 → item:/item!:/item!!:
  ✦ 既有物品の数量/归属/位置/性质が変化 → item:（変化部分のみ）
  ✦ 物品消耗/喪失/使い切り → item-:物品名
【何時書かない】
  ✗ 物品に変化なし → item行は一切出力禁止
  ✗ 物品は言及のみで状態変化なし → 書かない

【形式】
  新規獲得：item:emoji物品名(数量)|描述=持有者@精确位置（描述は省略可。贈り物/記念品など特別意味がある場合は描述追加）
  新規獲得(重要)：item!:emoji物品名(数量)|描述=持有者@精确位置（重要物品。描述必須：外観+機能+来源）
  新規獲得(关键)：item!!:emoji物品名(数量)|描述=持有者@精确位置（关键道具。描述は詳細必須）
  既有物品変化：item:emoji物品名(新数量)=新持有者@新位置（変化分のみ更新。|を書かなければ原描述を保持）
  消耗/喪失：item-:物品名

【字段級ルール】
  · 描述：物品の本質属性（外観/機能/来源）を記録。普通物品は省略可、重要/关键物品は初回必須
    ★ 外観特征（色・材質・サイズ等。後続描写の一貫性用）
    ★ 功能/用途
    ★ 来源（誰が渡したか/どう入手したか）
       - 示例（以下は示例のみ。本文へ原句流用禁止）：
         - 示例1：item!:🌹永生花束|深紅の永生玫瑰、黒リボン束ね。CがUへ贈った情人節ギフト=U@Uの部屋の机上
         - 示例2：item!:🎫幸运十连抽券|金色に輝く紙券。システム賞池で1回十連抽できる新人福利=U@空間指輪
         - 示例3：item!!:🏧位面货币自动兑换机|小型ATMのような外見で、各位面通貨を即時レートで交換可能=U@酒館吧台
  · 数量：単品は(1)/(1個)/(1本)などを書かない。計量単位がある場合のみ括弧記載（例(5斤)(1L)(1箱)）
  · 位置：必ず精確な固定地点
    ❌ 某人の足元/旁边、地板、桌子上 のような曖昧記述
    ✅ 酒館大廳地板、餐廳吧台上、家中厨房、背包里、Uの部屋の机上
  · 固定家具や建築設備を物品として計上しない
  · 一時貸与 ≠ 所有権移転


示例（麦酒ライフサイクル）：
  獲得：item:🍺陈酿麦酒(50L)|雑物間で見つけた酸味のある麦酒=U@酒館後厨食材柜
  量変：item:🍺陈酿麦酒(25L)=U@酒館後厨食材柜
  使い切り：item-:陈酿麦酒

═══ 【NPC】トリガー条件とルール ═══
形式：npc:名|外貌=性格@与{{user}}的关系~性别:值~年龄:值~种族:值~职业:值~生日:值
区切り記号：| 名前、= 外貌と性格、@ 関係、~ 拡張字段(key:value)

【何時書く】（いずれか満たすと該当NPCの npc: 行を出力）
  ✦ 初登場 → 完整形式。全字段+全~拡張字段（性别/年龄/种族/职业）必須
  ✦ 外貌の恒久変化（受傷痕、髪型変更、服飾変更など）→ 外貌字段のみ
  ✦ 性格転換（重大事件後の性格変化など）→ 性格字段のみ
  ✦ 与{{user}}関係定位の変化（客人→朋友など）→ 関係字段のみ
  ✦ そのNPCの新情報を獲得（以前不明の身長/体重など）→ 対応字段へ追記
  ✦ ~拡張字段自体の変化（职业変更など）→ 変化した~拡張字段のみ
【何時書かない】
  ✗ NPC在場だが新情報なし → npc: 行を書かない
  ✗ 一時退場後に再登場し、情報変化なし → 再記載禁止
  ✗ 同義語/略語で既存描写を書き換える → 厳禁
    ❌「肌肉发达/满身战斗伤痕」→「筋肉質/傷跡」（言い換え≠更新）
    ✅「肌肉发达/满身战斗伤痕/重伤」→「肌肉发达/满身战斗伤痕」（回復により古い状態を除去）

【増分更新示例】（NPC沃尔）
  初回：npc:沃尔|銀灰色披毛/緑眼/身長220cm/全身戦闘傷痕=寡黙な重装佣兵@{{user}}の最初の客人~性别:男~年龄:約35~种族:狼兽人~职业:佣兵
  関係のみ更新：npc:沃尔|=@{{user}}の男朋友
  外貌のみ追記：npc:沃尔|銀灰色披毛/緑眼/身長220cm/全身戦闘傷痕/左腕包帯
  性格のみ更新：npc:沃尔|=もはや寡黙ではない/時々微笑む
  職業のみ更新：npc:沃尔|~职业:退役佣兵
（注意：未変化字段と未変化~拡張字段は一切書かない。システムが原有データを自動保持）

【生日字段（任意拡張）】
  形式：~生日:yyyy/mm/dd または ~生日:mm/dd（年不明時は月日のみ）
  ⚠ 角色設定/人物描写で生日が明確に言及された場合のみ記入。推測や捏造は禁止
  ⚠ 明確な出典がない生日は一律記入しない。空欄のままユーザー入力へ

【関係描写ルール】
  対象名を必ず含み精確に：❌客人 ✅{{user}}的新访客 / ❌债主 ✅持有{{user}}借条的人 / ❌房东 ✅{{user}}の房东 / ❌男朋友 ✅{{user}}の男朋友 / ❌恩人 ✅救了{{user}}一命的人 / ❌霸凌者 ✅欺负{{user}}的人 / ❌暗恋者 ✅暗恋{{user}}的人 / ❌仇人 ✅被{{user}}杀掉了生父
  附属関係は所属NPC名も書く：✅伊凡的猎犬; {{user}}客人的宠物 / 伊凡的女朋友; {{user}}的客人 / {{user}}的闺蜜; 伊凡的妻子 / {{user}}的继父; 伊凡的父亲 / {{user}}的情夫; 伊凡的弟弟 / {{user}}的闺蜜; {{user}}の丈夫の情婦; 插足{{user}}与伊凡夫妻关系的第三者

═══ 【好感度】トリガー条件 ═══
NPC→{{user}}の好感度のみ記録（{{user}}自身は記録禁止）。1人1行、数値後注記禁止。

【何時書く】
  ✦ NPC初登場 → 関係に応じて初期値設定（陌生0-20/熟人30-50/朋友50-70/恋人70-90）
  ✦ 互动により実質変化 → affection:名=新総値
【何時書かない】
  ✗ 好感度変化なし → 書かない

═══ 【待办事项】トリガー条件 ═══
【何時書く（新規）】
  ✦ 新しい約定/計画/行程/任务/伏笔が発生 → agenda:日期|内容
  形式：agenda:订立日期|内容（相対時間は括弧で絶対日付併記）
  示例：agenda:2026/02/10|艾伦邀请{{user}}情人节晚上约会(2026/02/14 18:00)
【何時書く（完了削除）— 最重要】
  ✦ 待办が完了/失效/取消 → 必ず agenda-: で削除マーク
  形式：agenda-:待办内容（完了項目の关键词で自動削除可）
  示例：agenda-:艾伦邀请{{user}}情人节晚上约会
  ⚠ agenda:内容(完成) 形式は厳禁。必ず agenda-: 接頭辞を使う
  ⚠ 既存待办内容の重複記入禁止
【何時書かない】
  ✗ 既有待办に変化なし → 毎ターン重複出力禁止
  ✗ 待办完了 → agenda:に括弧で完了注記する方式は禁止。必ず agenda-:

═══ 時間形式ルール ═══
"Day 1"/"第X天" など曖昧形式は禁止。必ず具体的なカレンダー日付を使用。
- 現代：年/月/日 時:分（例 2026/2/4 15:00）
- 歴史：当該時代の日付（例 1920/3/15 14:00）
- 奇幻/架空：当該世界観カレンダー（例 霜降月第三日 黄昏）
- 時間を特定できない場合は、合理的で具体的な時刻を補って記述し、「不明」「某日」などの無意味な表現は禁止。

【毎ターン必須字段——1つでも欠けると不合格】
  ✅ time: ← 現在日付時刻
  ✅ location: ← 現在地点
  ✅ atmosphere: ← 雰囲気
  ✅ characters: ← 現在在場の全角色名（カンマ区切り、省略絶対不可）
  ✅ costume: ← 在場各角色ごとに1行の服装描写
  ✅ event: ← 重要程度|事件摘要

【NPC初登場時の追加必須——欠一不可】
  ✅ npc:名|外貌=性格@关系~性别:值~年龄:值~种族:值~职业:值~生日:值（既知時のみ生日を書く。不明なら書かない）

以上字段に「書いても書かなくてもよい」は存在しない。すべて強制。
