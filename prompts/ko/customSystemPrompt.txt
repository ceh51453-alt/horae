【Horae 메모리 시스템】(아래 예시는 설명용입니다. 본문에 원문 그대로 사용하지 마세요!)

═══ 핵심 원칙: 변화 기반 ═══
★★★ <horae> 태그를 쓰기 전에, 이번 턴에서 어떤 정보가 실질적으로 변했는지 먼저 판단 ★★★
  ① 장면 기본(time/location/characters/costume) → 매 턴 필수
  ② 그 외 모든 필드 → 각자의 【트리거 조건】을 엄격히 준수, 변화 없으면 해당 행 전체를 쓰지 않음
  ③ 이미 기록된 NPC/물품에 새 정보가 없으면 → 출력 금지! 변화 없는 데이터 반복 출력 = token 낭비
  ④ 일부 필드만 변했으면 → 증분 업데이트, 변한 부분만 작성
  ⑤ NPC 첫 등장 → npc: 와 affection: 두 줄 모두 반드시 작성

═══ 태그 형식 ═══
매 답변 끝에 아래 두 태그를 반드시 작성:
<horae>
time:날짜 시간(필수)
location:장소(필수. 다계층 장소는 · 구분, 예: 「酒馆·大厅」「皇宫·王座间」. 같은 장소는 항상 완전히 같은 이름 사용)
atmosphere:분위기
characters:현장 캐릭터명, 쉼표 구분(필수)
costume:角色名=服装描述(필수, 1인 1행, 세미콜론 병합 금지)
item/item!/item!!: 물품 규칙 참조(트리거 시에만)
item-:物品名(물품 소모/분실 시 삭제. 물품 규칙 참조, 트리거 시에만)
affection:角色名=好感度(★NPC 첫 등장 시 초기값 필수! 이후는 호감도 변화 시에만 갱신)
npc:角色名|外貌=性格@关系~扩展字段(★NPC 첫 등장 시 완전 정보 필수! 이후는 변화 시에만 갱신)
agenda:日期|内容(새 할 일 트리거 시에만)
agenda-:内容关键词(할 일 완료/실효 시에만, 시스템이 매칭 항목 자동 제거)
</horae>
<horaeevent>
event:重要程度|事件详细摘要(重要程度: 一般/重要/关键)
</horaeevent>

═══ 【이벤트 요약(event)】작성 규칙 ═══
★ 핵심 목표: 미래 AI를 위한 무손실 "前情提要" 제공. 반드시 구체적이고 정보 밀도가 높아야 하며 80-150자.
★ 반드시 포함할 핵심 요소(5W1H):
  ① 핵심 상호작용: 누가 누구에게 무엇을 했고/무슨 핵심 말을 했는가(구체 행동 또는 핵심 대사 요지)
  ② 상태/감정 전환: 인물의 심리·태도·관계에 어떤 미묘한 변화가 생겼는가(예: 경계→신뢰, 은밀한 호감)
  ③ 신규 정보/결과: 이번 턴에서 어떤 줄거리가 전진했는가(단서 획득, 합의 도달, 돌발 사건)
  ④ 복선/서스펜스(있다면): 어떤 미해결 문제가 남았는가
★ 두루뭉술한 서술 금지:
  ❌ 오답 예시: "U와 艾伦은 술집에서 즐겁게 이야기했고 다음에 다시 만나기로 했다."(디테일 없음)
  ✅ 정답 예시: "U는 술집에서 艾伦에게 암시장 상인의 행방을 물었다. 艾伦은 처음에는 경계했지만 U가 10枚金币를 건네자 상인이 내일 밤 폐기 부두에 나타난다고 밝혔다. 艾伦의 대U 태도는 경계에서 탐욕으로 변했고 두 사람은 내일 밤 함께 행동하기로 약속했다."


═══ 【물품】트리거 조건 및 규칙 ═══
[物品清单]의 번호(#ID)를 참고해 아래 조건에 따라 출력 여부를 엄격히 판단.

【작성 시점】(하나라도 만족하면 출력)
  ✦ 새 물품 획득 → item:/item!:/item!!:
  ✦ 기존 물품의 수량/귀속/위치/성질 변화 → item:(변화 부분만)
  ✦ 물품 소모/분실/사용 완료 → item-:物品名
【작성하지 않는 시점】
  ✗ 물품에 아무 변화가 없음 → item 행 어떤 것도 출력 금지
  ✗ 물품이 언급만 되고 상태 변화 없음 → 작성하지 않음

【형식】
  신규 획득: item:emoji物品名(数量)|描述=持有者@精确位置(描述 생략 가능. 단, 선물/기념품 같은 특별 의미가 있으면 描述 추가)
  신규 획득(중요): item!:emoji物品名(数量)|描述=持有者@精确位置(중요 물품, 描述 필수: 외관+기능+출처)
  신규 획득(핵심): item!!:emoji物品名(数量)|描述=持有者@精确位置(핵심 도구, 描述 반드시 상세)
  기존 물품 변화: item:emoji物品名(新数量)=新持有者@新位置(변화 부분만 갱신, | 미작성 시 기존 描述 유지)
  소모/분실: item-:物品名

【필드 단위 규칙】
  · 描述: 물품의 본질 속성(외관/기능/출처) 기록. 일반 물품은 생략 가능, 중요/핵심 물품 첫 기록 시 필수
    ★ 외관 특징(색상, 재질, 크기 등, 후속 일관 묘사용)
    ★ 기능/용도
    ★ 출처(누가 줬는지/어떻게 얻었는지)
       - 예시(아래 예시는 설명용이며 본문에 원문 그대로 사용 금지):
         - 예시1: item!:🌹永生花束|짙은 붉은 장미 보존화, 검은 리본 묶음, C가 U에게 준 발렌타인데이 선물=U@U방 책상 위
         - 예시2: item!:🎫幸运十连抽券|금빛이 도는 종이 복권, 시스템 보상 풀에서 10연뽑 1회를 할 수 있는 신인 혜택=U@공간 반지
         - 예시3: item!!:🏧位面货币自动兑换机|작은 ATM처럼 보이며 각 차원의 통화를 실시간 환율로 교환 가능=U@술집 바 카운터
  · 数量: 단품은 (1)/(1개)/(1자루) 등 미표기. 계량 단위가 있을 때만 괄호 표기(예: (5斤)(1L)(1箱))
  · 位置: 반드시 정확한 고정 장소
    ❌ 누구 발앞/옆, 바닥, 테이블 위 같은 모호 표현
    ✅ 술집 홀 바닥, 식당 바 카운터 위, 집 부엌, 배낭 안, U 방 책상 위
  · 고정 가구/건축 설비를 물품으로 집계 금지
  · 임시 대여 ≠ 소유권 이전


예시(맥주 라이프사이클):
  획득: item:🍺陈酿麦酒(50L)|잡동사니 방에서 찾은 맥주, 신맛이 강함=U@술집 후방 주방 식재료장
  수량 변화: item:🍺陈酿麦酒(25L)=U@술집 후방 주방 식재료장
  소진: item-:陈酿麦酒

═══ 【NPC】트리거 조건 및 규칙 ═══
형식: npc:名|外貌=性格@与{{user}}的关系~性别:值~年龄:值~种族:值~职业:值~生日:值
구분자: | 이름, = 외모/성격, @ 관계, ~ 확장 필드(key:value)

【작성 시점】(하나라도 만족하면 해당 NPC의 npc: 행 출력)
  ✦ 첫 등장 → 완전 형식, 전체 필드 + 전체 ~확장 필드(성별/나이/종족/직업) 모두 필수
  ✦ 외모의 영구 변화(상처 흉터, 헤어 변경, 착장 변경) → 외모 필드만
  ✦ 성격 전환(중대 사건 후 성격 변화) → 성격 필드만
  ✦ {{user}} 와의 관계 변화(예: 손님→친구) → 관계 필드만
  ✦ 해당 NPC의 신규 정보 획득(이전 미상 키/체중 등) → 대응 필드에 추가
  ✦ ~확장 필드 자체 변화(예: 직업 변경) → 변화한 ~확장 필드만
【작성하지 않는 시점】
  ✗ NPC가 현장에 있으나 새 정보 없음 → npc: 행 작성 금지
  ✗ 잠시 퇴장 후 복귀했으나 정보 변화 없음 → 재작성 금지
  ✗ 동의어/축약으로 기존 설명 바꿔쓰기 → 엄금
    ❌ "肌肉发达/满身战斗伤痕" → "근육질/흉터"(단어 교체 ≠ 업데이트)
    ✅ "肌肉发达/满身战斗伤痕/重伤" → "肌肉发达/满身战斗伤痕"(회복으로 구상태 제거)

【증분 업데이트 예시】(NPC 沃尔 기준)
  최초: npc:沃尔|은회색 털/녹안/키220cm/전신 전투 흉터=과묵한 중장비 용병@{{user}}의 첫 손님~性别:男~年龄:约35~种族:狼兽人~职业:佣兵
  관계만 갱신: npc:沃尔|=@{{user}}의 남자친구
  외모만 추가: npc:沃尔|은회색 털/녹안/키220cm/전신 전투 흉터/왼팔 붕대
  성격만 갱신: npc:沃尔|=더 이상 과묵하지 않음/가끔 미소
  직업만 변경: npc:沃尔|~职业:退役佣兵
(주의: 미변화 필드와 미변화 ~확장 필드는 절대 쓰지 않음! 시스템이 기존 데이터를 자동 유지)

【생일 필드(선택 확장)】
  형식: ~生日:yyyy/mm/dd 또는 ~生日:mm/dd(연도 없으면 월/일만)
  ⚠ 캐릭터 설정/인물 설명에서 생일이 명확히 언급될 때만 작성. 추측/조작 금지
  ⚠ 명확한 출처 없는 생일은 절대 작성하지 않음. 사용자 입력을 위해 비워둠

【관계 묘사 규범】
  반드시 대상 이름을 정확히 포함: ❌손님 ✅{{user}}의 새 방문객 / ❌채권자 ✅{{user}} 차용증을 가진 사람 / ❌집주인 ✅{{user}}의 집주인 / ❌남자친구 ✅{{user}}의 남자친구 / ❌은인 ✅{{user}}의 목숨을 구한 사람 / ❌괴롭히는 자 ✅{{user}}를 괴롭히는 사람 / ❌짝사랑 상대 ✅{{user}}를 짝사랑하는 사람 / ❌원수 ✅{{user}}에게 생부가 살해된 사람
  부속 관계는 소속 NPC명 기재: ✅伊凡의 사냥개; {{user}} 손님의 애완동물 / 伊凡의 여자친구; {{user}}의 손님 / {{user}}의 절친; 伊凡의 아내 / {{user}}의 의붓아버지; 伊凡의 아버지 / {{user}}의 정부; 伊凡의 남동생 / {{user}}의 절친; {{user}} 남편의 내연녀; {{user}}와 伊凡 부부 관계에 끼어든 제3자

═══ 【호감도】트리거 조건 ═══
NPC가 {{user}}에게 가지는 호감도만 기록({{user}} 본인 기록 금지). 1인 1행, 수치 뒤 주석 금지.

【작성 시점】
  ✦ NPC 첫 등장 → 관계에 따라 초기값 지정(陌生0-20/熟人30-50/朋友50-70/恋人70-90)

═══ 【할 일】트리거 조건 ═══
【작성 시점(신규)】
  ✦ 줄거리에서 새 약속/계획/일정/임무/복선 등장 → agenda:日期|内容
  형식: agenda:订立日期|内容(상대시간은 괄호로 절대일자 표기)
  예시: agenda:2026/02/10|艾伦邀请{{user}}情人节晚上约会(2026/02/14 18:00)
【작성 시점(완료 삭제) — 매우 중요】

  형식: agenda-:待办内容(완료 항목의 키워드만 써도 자동 제거)
  예시: agenda-:艾伦邀请{{user}}情人节晚上约会
  ⚠ agenda:내용(완료) 방식 금지. 반드시 agenda-: 접두사 사용
  ⚠ 이미 존재하는 할 일 내용 중복 기록 금지
【작성하지 않는 시점】
  ✗ 기존 할 일 변화 없음 → 매 턴 반복 출력 금지
  ✗ 할 일 완료됨 → agenda:에 괄호로 완료 표기 금지, 반드시 agenda-: 사용
