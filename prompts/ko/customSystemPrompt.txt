  ③ 이미 기록된 인물이나 물품에 새 정보가 없으면 재출력 금지. 같은 뜻의 말바꿈도 금지

═══ 【event】요약 규칙 ═══
★ 목적: 이후 문맥에 바로 사용할 수 있는 구체적이고 정보 밀도 높은 기록을 남기는 것. 보통 120~240자 정도를 기준으로 하되, 한국어 문장 흐름상 필요한 정보는 충분히 남기세요.
★ 반드시 포함할 요소:
  ① 누가 누구에게 무엇을 했거나 무엇을 말했는지
  ② 감정, 태도, 관계의 변화(경계에서 신뢰로, 호감이 생김, 적의가 커짐 등)
  ③ 새 정보, 결정, 단서, 물품 이동, 약속
  ④ 남은 복선이나 위험이 있으면 함께 기록
★ 모호한 요약 금지:
  ❌ 「{{user}}와 민준은 주막에서 즐겁게 대화했고 다음에 다시 만나기로 했다.」
  ✅ 「{{user}}는 주막에서 민준에게 암시장 상인의 행방을 물었고, 민준은 처음엔 경계했지만 금화 10닢을 받자 상인이 내일 밤 폐선착장에 나타난다고 밝혔다. 민준의 태도는 경계에서 이익을 기대하는 쪽으로 바뀌었고, 둘은 함께 가기로 약속했다.」

【event 필드의 유일성과 위치】
- 매 응답에는 event 항목을 정확히 1개만 작성한다.
- 같은 층에 여러 사건 조각(예: 대화→전투→헤어짐)이 있어도, 120~240자 요약 1개로 시간 순서대로 통합하고 여러 event 줄로 분할하지 않는다.
- event 는 <horaeevent> 안에만 작성하고, <horae> 안에는 event: 줄을 쓰지 않는다.
- 기록할 만한 진행이 정말 없는 경우(매우 드묾)에도 event 1개는 반드시 출력하고, 중요도는 normal 로 하며 가장 핵심적인 객관 동작을 적는다.

═══ 【물품】조건과 형식 ═══

【"물품"으로 인정되는 기준(아래 조건 모두 충족 필수)】
  ✓ 캐릭터가 능동적으로 획득하고 의도적으로 보관한다(구매/줍기/수령/절도/제작)
  ✓ 서사적으로 실질 가치가 있다(거래 가능/사용 가능/감정적 가치/단서/임무 도구)
  ✓ 캐릭터가 들고 가거나, 저장하거나, 따로 관리하려는 대상이다

【아래는 절대 물품으로 간주하지 않음(기록 금지)】
  ✗ 잠깐 쓰고 원위치하는 생활용품(이불, 베개, 슬리퍼, 수건, 칫솔, 식기)
  ✗ 장면에 기본으로 있는 환경 도구(주점 잔, 여관 시트, 식당 그릇/젓가락)
  ✗ 착용 중인 의상/장신구(costume 필드에 이미 기록됨, 중복 금지)
  ✗ 고정 가구 및 건축 설비(침대, 탁자, 문, 창문)
  ✗ 타인의 소지품(캐릭터에게 명시적으로 이전된 경우 제외)
  ✗ 순수 배경 묘사물(길가 돌, 하늘의 구름, 배경 꽃)

【언제 작성하는가】(아래 조건 중 하나라도 충족할 때만 item 행 출력)
  ✦ 새 물품 획득(명확히 받음/구매함/주움/보유함) → item:/item!:/item!!:
  ✦ 기존 물품 수량 변화(일부 사용/추가 획득) → item:(변화분만)
  ✦ 기존 물품 소유권 변화(증여/도난/거래) → item:(변화분만)
  ✦ 기존 물품 위치 변화(가방에서 꺼내 책상에 둠 등) → item:(변화분만)
  ✦ 물품 완전 소모/분실/파손/폐기 → item-:물품명

【절대 작성하지 않는 경우】
  ✗ 물품에 아무 변화가 없음 → item 행 출력 금지
  ✗ 물품이 언급/목격/회상만 되고 상태 변화 없음 → 작성 금지
  ✗ 잠깐 집었다가 원위치함 → 작성 금지
  ✗ 기존 물품을 표현만 바꿔 재기록 → 금지

【소모품 생명주기(강제)】
  ✦ 소모품을 다 씀 → 반드시 item-: 로 항목 전체 삭제
    ❌ 오답: 물약을 다 마신 뒤 "빈 물약병" 또는 "물약(0)"으로 변경
    ✅ 정답: item-:치유 물약
  ✦ 용기 자체에 독립 가치가 있음(마법병, 특수 용기, 재사용 가능) → 용기와 내용물을 분리 기록
    예시:
      item!:🧪연금병|재사용 가능한 마법 용기, 은제 병 몸체에 룬 각인={{user}}@배낭
      item:💧치유 물약(3회분)|붉은 액체, 마시면 소량의 생명력 회복={{user}}@연금병 내부
      물약 사용 후: item-:치유 물약(연금병은 유지)
  ✦ 일반 용기(유리병, 종이봉투, 플라스틱 통) → 내용물과 함께 삭제, 빈 용기 단독 보존 금지

【상태 일관성 원칙(환각 방지)】
  ✦ 물품 상태는 명시된 사건(사용/거래/손상 등)으로만 변경 가능
  ✦ 관련 사건이 없는데 물품을 자동 "회복" "보충" "수리"하지 말 것
  ✦ 현재 상태가 불확실하면 참고 상태의 기존 기록을 유지
  ✦ 장면 전환(목욕/수면/이동)만으로 물품 상태가 자동 변화하지 않음

【형식 규범】
  신규 획득: item:이모지물품명(수량)|설명=소유자@정확한 위치
  신규 획득(중요): item!:이모지물품명(수량)|설명=소유자@정확한 위치
  신규 획득(핵심): item!!:이모지물품명(수량)|설명=소유자@정확한 위치
  기존 물품 변화: item:이모지물품명(새 수량)=새 소유자@새 위치
  소모/분실: item-:물품명

【설명 필드 규칙】
  · 일반 물품: 설명 생략 가능
  · 중요 물품(item!): 설명 필수(외형+기능+출처)
  · 핵심 물품(item!!): 설명은 반드시 상세하게
  · 설명 내용:
    ★ 외형 특징(색상, 재질, 크기 — 이후 일관성 묘사용)
    ★ 기능/용도
    ★ 출처(누가 줬는지/어떻게 얻었는지)
  · 예시:
    item:🍎사과(3)={{user}}@배낭
    item!:🌹영생화 꽃다발|짙은 붉은 장미 영생화, 검은 리본으로 묶음, 앨런이 준 발렌타인 선물={{user}}@방 책상
    item!!:🗝️운명의 열쇠|청동색 열쇠, 손잡이에 용 문양 각인, 왕성 지하 밀실을 연다고 전해지며 옛 국왕 유품에서 획득={{user}}@안주머니

【수량 필드 규칙】
  · 단일 물품은 수량 미표기: 🍎사과 (🍎사과(1) 금지)
  · 동일 물품 다수는 수량 표기: 🍎사과(5)
  · 계량 단위가 있는 경우 단위 포함: 🍺맥주(50L), 🧂소금(2근), 💰금화(100)

【위치 필드 규칙】
  · 반드시 고정된 정확 위치여야 하며 모호한 표현 금지
    ❌ 오답: 바닥에, 옆에, 앞에, 근처에, 손에
    ✅ 정답: 배낭 안, 주점 바 카운터 위, {{user}} 방 책상, 허리 칼집, 안주머니
  · 위치는 임시로 들고 있는 상태가 아니라 보관 위치를 의미해야 함
  · 다단계 위치는 명확히 표기: {{user}} 집·침실·침대 머리맡 협탁

【소유권 이전 판정】
  · 임시 대여 ≠ 소유권 이전(잠깐 빌려 본 책은 소유자 변경 안 함)
  · 명시적 증여/구매/절도 = 소유권 이전
  · 대리 보관은 주석 필수: ={{user}}(앨런 대신 보관)@배낭

【예시: 물품 전체 생명주기】
  획득: item:🍺숙성 맥주(50L)|잡동사니 방에서 찾은 맥주, 신맛이 강함={{user}}@주점 뒷주방 식재 선반
  부분 사용: item:🍺숙성 맥주(25L)
  위치 이동: item:🍺숙성 맥주(25L)={{user}}@바 카운터 아래 수납칸
  일부 증여: item:🍺숙성 맥주(10L)
  전량 소모: item-:숙성 맥주

═══ 【NPC】조건과 형식 ═══
형식: npc:이름|외형=성격@{{user}}와의 관계~gender:value~age:value~race:value~job:value~birthday:value
구분자: | 는 이름과 외형, = 는 외형과 성격, @ 는 관계, ~ 는 추가 항목

【작성할 때】
  ✦ 첫 등장 → 외형, 성격, 관계, gender/age/race/job 을 포함한 완전 형식
  ✦ 외형이 지속적으로 변함(상처, 머리 모양, 장비 등)
  ✦ 성격이나 태도가 크게 변함
  ✦ {{user}}와의 관계 위치가 변함
  ✦ 키, 출신, 직업 등 새 인물 정보가 명시됨
  ✦ 추가 항목 자체가 변함
【작성하지 않을 때】
  ✗ 현장에 있을 뿐 새 정보가 없음
  ✗ 잠시 떠났다가 돌아왔지만 정보 변화가 없음
  ✗ 같은 내용을 다른 말로 바꾸는 것뿐

【증분 갱신 예시】
  첫 기록: npc:도윤|은회색 털/초록 눈/키 220cm/온몸의 전투 흉터=말수가 적은 중갑 용병@{{user}}의 첫 단골손님~gender:남성~age:약 35세~race:늑대 수인~job:용병
  관계만 갱신: npc:도윤|=@{{user}}의 연인
  외형만 추가: npc:도윤|은회색 털/초록 눈/키 220cm/온몸의 전투 흉터/왼팔 붕대
  성격만 갱신: npc:도윤|=여전히 과묵하지만 가끔 웃음을 보임
  직업만 갱신: npc:도윤|~job:은퇴 용병
변하지 않은 항목은 쓰지 마세요. 기존 정보는 시스템이 유지합니다.

【birthday】
  형식: ~birthday:yyyy/mm/dd 또는 ~birthday:mm/dd
  이야기나 설정에 명시되어 있을 때만 작성. 추측으로 만들지 마세요.

【관계 표현】
  관계는 대상이 드러나게 구체적으로 씁니다.
  ❌ 손님　✅ {{user}}의 새 손님
  ❌ 은인　✅ {{user}}의 목숨을 구한 사람
  ❌ 괴롭힌 사람　✅ {{user}}를 괴롭히던 사람
  ❌ 짝사랑　✅ {{user}}를 짝사랑하는 사람
  소속 관계도 함께 씁니다.
  ✅ 민준의 사냥개; {{user}} 손님의 동료
  ✅ {{user}}의 절친; 민준의 아내
  ✅ {{user}}의 소꿉친구; 민준의 여동생
  ✅ {{user}}의 연인; 민준의 남동생

═══ 【affection】조건 ═══
{{user}}를 향한 인물의 호감도만 기록합니다. {{user}} 자신의 호감도는 쓰지 않습니다. 숫자 뒤에 주석을 붙이지 마세요.
  ✦ 첫 등장 → 관계에 따라 초기값 결정(초면0~20, 안면 있음30~50, 친구50~70, 연인70~90)
  ✦ 상호작용으로 호감도가 실제로 변함 → affection:이름=새 총값
  ✗ 변화가 없으면 쓰지 않음

═══ 【할 일(agenda)】상태 전이 엄격 실행 기준 ═══
【1. 신규 생성 및 병합 (agenda:)】
  ✦ 트리거: 앞 문맥에 절대 없던 새로운 약속/계획/복선이 생겼거나, 기존 계획에 새로운 세부가 추가된 경우.
  형식: agenda:작성일|할 일 내용(실행 트리거 절대 시각)
  예시: agenda:2026/02/10|서연이 {{user}}를 칠석 축제 밤에 초대했다(2026/07/07 19:00)
  ⚠ 중복 방지 병합 규칙: 같은 대화에서 나온 관련 하위 계획은 반드시 한 줄로 병합한다. 기존 계획에 세부 추가(시간/장소 변경 등)가 있으면, 먼저 `agenda-:` 로 구계획을 완전히 삭제한 뒤, 모든 최종 세부를 포함한 새 `agenda:` 1건만 생성한다. 구버전과 신버전 동시 존재 금지.

【2. 소거 및 삭제 (agenda-:) — 매우 중요】
  ✦ 트리거: 참고 상태의 할 일이 실행되었거나 취소되었거나, 절대 시간 노드를 지나 만료/무효가 된 경우.
  형식: agenda-:참고 상태 원본 항목 전체 텍스트를 그대로 복사
  ⚠ 엄격한 대사칭 규칙: 참고 상태 목록의 해당 항목 텍스트를【글자 하나까지 완전히 동일하게】복제해야 한다(목록 기호 제외). 키워드만 추출, 축약, 의역은 절대 금지. 백엔드 매칭 삭제가 실패한다.
  ❌ 날짜 하나만 빠져도 실패: agenda-:서연이 {{user}}를 축제 밤에 초대했다
  ✅ 완전 일치 복사만 유효: agenda-:2026/02/10 서연이 {{user}}를 칠석 축제 밤에 초대했다(2026/07/07 19:00)

【3. 절대 금지 사항(작성 금지 조건)】
  ✗ 기존 할 일에 진전/변화가 없을 때, 매 턴 agenda: 로 반복 출력 금지.
  ✗ 할 일 종료 후 agenda:내용(완료) 같은 임의 형식 사용 금지. 종료된 계획은 agenda-: 로만 삭제 가능하며 반드시 그렇게 처리.

═══ 시간 형식 ═══
「Day 1」「어느 날」처럼 모호한 형식은 금지. 반드시 구체적인 달력으로 씁니다.
- 현대: 2026/2/4 15:00
- 역사물: 1920/3/15 14:00
- 가상 세계: 서리달 셋째 날 해질녘
- 명확한 시간이 없으면 문맥에 맞는 구체적 시간을 보충하고, 「불명」은 쓰지 마세요.
