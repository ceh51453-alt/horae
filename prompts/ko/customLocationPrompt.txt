═══ 【장면 기억】트리거 조건 ═══
형식: scene_desc:~에 위치. 해당 장소의 고정 물리 특징 설명(50-150자)
장면 기억은 장소의 핵심 레이아웃과 영구적 특징(건축 구조, 고정 가구, 공간 특성)을 기록하여 턴 간 장면 묘사의 일관성을 유지합니다.

【장소/“~에 위치” 형식】★★★ 계층 규칙 엄수 ★★★
  · 설명 시작에서 먼저 "위치"를 써서 직속 상위 기준의 방위를 표시하고, 이어서 해당 장소 자체의 물리 특징을 작성
  · 하위 장소(지명에 · 구분자 포함): "위치"에는 부모 건물 내부 방위만 작성(몇 층/어느 방향). 부모의 외부 지리 정보는 절대 금지
  · 부모/최상위 장소: "위치"에 외부 지리 정보(어느 대륙, 어느 숲 근처 등)를 쓰는 것은 여기만 허용
  · 시스템이 부모 설명을 AI에 자동 전달하므로, 하위 장소는 부모 정보를 반복하지 않아야 함
    ✓ 이름 없는 술집·객실203 → scene_desc:2층 동쪽에 위치. 모서리 방, 채광 좋음, 1인용 목재 침대는 벽쪽, 창문은 동향
    ✓ 이름 없는 술집·홀 → scene_desc:1층에 위치. 천장이 높은 목재 공간, 중앙에 긴 바 카운터, 둥근 테이블 여러 개가 흩어져 있음
    ✓ 이름 없는 술집 → scene_desc:OO 대륙 북부 XX 숲 가장자리에 위치. 2층 목석 구조, 1층은 홀과 바, 2층은 객실 구역
    ✗ 이름 없는 술집·객실203 → scene_desc:OO 대륙 북부 XX 숲 가장자리의 이름 없는 술집 2층에 위치... (❌ 하위 장소는 부모 외부 지리 정보 작성 금지)
    ✗ 이름 없는 술집·홀 → scene_desc:숲 가장자리의 이름 없는 술집 1층에 위치... (❌ 동일)
【지명 규칙】
  · 다층 장소는 · 로 구분: 건물·구역(예: "이름 없는 술집·홀", "왕궁·지하감옥")
  · 동일 장소는 항상 상단 [장면|...] 과 완전히 동일한 이름 사용. 축약/개작 금지
  · 서로 다른 건물의 동명이구역은 독립 기록(예: "이름 없는 술집·홀"과 "왕궁·홀"은 다른 장소)
【작성 시점】
  ✦ 새로운 장소에 처음 도착 → scene_desc 반드시 작성, 해당 장소의 고정 물리 특징 설명
  ✦ 장소에 영구적 물리 변화 발생(파괴/리모델링 등) → 업데이트된 scene_desc 작성
【작성하지 않는 시점】
  ✗ 이미 기록된 장소로 돌아왔고 물리 변화 없음 → 작성하지 않음
  ✗ 계절/날씨/분위기 변화 → 작성하지 않음(임시 변화이며 고정 특징이 아님)
【묘사 규칙】
  · 고정/영구 물리 특징만 작성: 공간 구조, 건축 재질, 고정 가구, 창문 방향, 상징 장식
  · 임시 상태는 작성 금지: 현재 조명, 날씨, 인파, 계절 장식, 임시 배치 물품
  · 장면 기억 원문을 본문에 그대로 복사 금지. 배경 참고로 사용하고 현재 시간/날씨/광선/시점에 맞게 재서술
  · 상단 [장면기억|...] 은 해당 장소의 시스템 기록 특징. 장면 묘사 시 핵심 요소는 유지하고, 시간/계절/전개에 따라 세부 변화는 자유롭게 보완
